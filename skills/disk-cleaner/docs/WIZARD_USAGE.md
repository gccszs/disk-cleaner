# 交互式向导使用指南

## 概述

`interactive_wizard.py` 是一个完整的交互式磁盘清理向导，提供用户友好的分步界面。

## 功能特性

### 1. 欢迎页面
- 显示功能介绍
- 说明安全性承诺
- 用户确认后继续

### 2. 扫描模式选择

#### 快速扫描 (1秒)
- 扫描常见临时文件位置
- 文件数限制：10,000
- 时间限制：1秒
- 适合日常清理

#### 标准扫描 (2分钟)
- 全面扫描当前目录
- 包含子目录
- 文件数限制：100,000
- 时间限制：120秒
- 推荐模式

#### 深度扫描 (无限制)
- 扫描所有文件
- 使用配置默认值（1M文件，25秒）
- 适合彻底清理

#### 渐进式扫描
- 使用缓存增量扫描
- 后续扫描更快
- 适合定期清理

#### 自定义扫描
- 用户指定参数
- 最大文件数
- 最大扫描时间

### 3. 分类展示和安全级别

#### 安全级别 (OK)
- 临时文件
- 缓存文件
- 默认选中

#### 警告级别 (!)
- 开发工具索引
- 构建产物
- 需手动勾选

#### 危险级别 (X)
- 用户文档
- 重要文件
- 需逐个确认

### 4. 多级确认流程

#### 第一级：清理摘要
- 显示按风险分组的文件
- 总文件数和大小
- 风险级别说明

#### 第二级：危险操作确认
- 如果选择危险级别文件
- 需要输入 "I UNDERSTAND"

#### 第三级：最终确认
- 需要输入 "YES"
- 明确警告：永久删除

### 5. 执行清理
- 显示进度
- 创建备份
- 记录日志
- 显示结果

## 使用方法

### 基本用法

```bash
# 清理当前目录
python scripts/interactive_wizard.py

# 清理指定目录
python scripts/interactive_wizard.py /path/to/directory

# Windows
python scripts\interactive_wizard.py C:\Users\Username\Downloads

# Linux/macOS
python scripts/interactive_wizard.py ~/Downloads
```

## 集成组件

向导集成了以下现有组件：

- `InteractiveCleanupUI` - 交互式展示和选择
- `ProgressBar` - 进度条显示
- `IndeterminateProgress` - 不定长进度显示
- `DirectoryScanner` - 目录扫描
- `SmartCleanupEngine` - 智能清理引擎
- `FileClassifier` - 文件分类器
- `DuplicateFinder` - 重复文件检测
- `SafetyChecker` - 安全检查

## 安全特性

1. **多重确认**
   - 每个删除操作都需要明确确认
   - 危险操作需要额外确认步骤

2. **备份机制**
   - 可选备份功能
   - 删除前创建备份

3. **日志记录**
   - 所有删除操作记录到日志
   - 日志位置：`~/.disk-cleaner/logs/cleanup.log`

4. **权限检查**
   - 验证文件访问权限
   - 跳过无法访问的文件

5. **文件锁检测**
   - 检测正在使用的文件
   - 避免删除锁定文件

## ASCII 安全输出

脚本输出使用 ASCII 字符，确保在所有终端环境下正常显示：

- 使用 `[OK]`、`[!]`、`[X]` 代替 emoji
- 风险级别使用文本标识符
- 进度条使用 ASCII 字符

Agent 汇报时可以使用 emoji 提供更好的视觉体验。

## 配置

向导使用 `~/.disk-cleaner/config.yaml` 配置文件：

```yaml
# 受保护的路径和文件
protected:
  paths:
    - "important-project/"
    - "database/"
  patterns:
    - "*.database"
    - "*.db"

# 清理规则
rules:
  - name: "旧日志"
    pattern: "*.log"
    category: "日志"
    risk: "safe"
    age_threshold: 60

# 安全设置
safety:
  check_file_locks: true
  verify_permissions: true
  backup_before_delete: true

# 扫描设置
scan:
  use_incremental: true
  cache_ttl: 7  # 天
```

## 故障排除

### 问题：扫描速度慢

**解决方案：**
- 使用 "快速扫描" 模式
- 使用 "渐进式扫描" 启用缓存
- 减少扫描范围

### 问题：权限错误

**解决方案：**
- 使用管理员/root权限运行
- 检查配置文件中的受保护路径
- 跳过系统目录

### 问题：编码错误

**解决方案：**
- 确保终端支持 UTF-8
- Windows: `chcp 65001`
- Linux: `export LANG=en_US.UTF-8`

## 最佳实践

1. **首次使用**
   - 从 "快速扫描" 开始
   - 只选择 "安全级别" 文件
   - 查看日志确认结果

2. **定期清理**
   - 使用 "渐进式扫描"
   - 启用缓存提升性能
   - 设置定时任务自动运行

3. **彻底清理**
   - 使用 "深度扫描"
   - 仔细检查 "危险级别" 文件
   - 创建备份以防万一

4. **开发环境**
   - 使用 "自定义扫描"
   - 排除 `node_modules` 等大型目录
   - 定期清理构建产物
